01 — Fundamentos
// 01.01

O que é um Sistema Distribuído?

Sistema distribuído é um conjunto de computadores que são independentes e trabalham juntos para que se pareçam como um sistema unificado. Os computadores se comunicam através da rede compartilhando dados para executar tarefas de forma ordenada. O sistema tem que ter transparência onde o usuário não perceba que existem vários computadores trabalhando por trás, e, escalabilidade onde é possível aumentar a capacidade do sistema adicionando mais maquinas, porem sem precisar alterar toda a estrutura existente.

🔗 Não depende de uma única máquina

Em vez de tudo rodar em um só computador, as tarefas são divididas entre vários nós de forma cooperativa.

🛡️ Maior resistência a falhas

Se um computador parar de funcionar, os outros podem continuar o trabalho, evitando que o sistema inteiro caia.

// 01.02

Três Tipos de Sistemas Distribuídos

Os sistemas distribuídos se dividem em categorias conforme sua finalidade e contexto de uso.

⚙️

Computação Distribuída

Focados em dividir tarefas pesadas entre várias máquinas para ganhar desempenho e velocidade de processamento.

ex: pesquisas científicas com big data
🏦

Informação Distribuída

Permitem que diferentes sistemas compartilhem dados entre si de forma transparente e consistente.

ex: sistemas bancários multi-agências
🏠

Embarcados Distribuídos

Presentes em dispositivos conectados que trabalham juntos, formando redes de sensores e atuadores físicos.

ex: casas inteligentes / IoT
02 — Arquitetura
// 02.01

Cliente–Servidor vs Peer-to-Peer

São os dois principais modelos de comunicação em sistemas distribuídos, com características, vantagens e limitações distintas.

🖥️ Cliente–Servidor

Existe um servidor central que fornece serviços, e vários clientes que fazem requisições.

✔Mais fácil de organizar, controlar e manter
✘Se o servidor parar, os clientes ficam sem acesso

🔄 Peer-to-Peer (P2P)

Não existe servidor central — cada computador pode enviar e receber dados livremente.

✔Não depende de um ponto único, mais resistente a falhas
✘Mais difícil de controlar e pode ter problemas de segurança
// 02.02

Supercomputação

Supercomputação é o uso de computadores extremamente potentes para resolver problemas muito complexos que um computador comum não conseguiria processar em tempo viável.

Áreas de aplicação: Previsão do tempo — para calcular mudanças climáticas com mais precisão; Ciência e pesquisa — simulações de moléculas, testes de medicamentos e estudos espaciais.

03 — Processos & Comunicação
// 03.01

Processos e Concorrência

Um processo é um programa em execução — ou seja, algo que está rodando naquele momento no sistema.

Em sistemas distribuídos, vários processos acontecem ao mesmo tempo (concorrência). Isso pode causar problemas quando dois ou mais processos tentam acessar o mesmo recurso simultaneamente.

⚠ Condição de Corrida: Os processos "disputam" o mesmo dado ao mesmo tempo, e o resultado final pode dar errado — o estado do sistema fica inconsistente por falta de coordenação.

// 03.02

Comunicação entre Processos

Nos sistemas distribuídos, os processos precisam trocar informações pela rede para funcionar corretamente. Isso é feito através de envio e recebimento de mensagens.

Sockets
RPC — Remote Procedure Call
  • Sockets — permitem a comunicação direta entre máquinas pela rede, de forma simples e eficiente.
  • RPC — permite que um programa execute funções em outro computador como se fosse local, abstraindo a complexidade da comunicação em rede.
// 03.03

Nomeação (Naming)

Nomeação é a forma de identificar recursos dentro do sistema — como servidores, arquivos ou serviços — usando nomes em vez de endereços complexos.

Por que é importante? Facilita muito o uso do sistema: em vez de decorar um endereço de rede, você usa um nome simples e o sistema resolve o resto automaticamente — como o DNS faz com os sites da internet.

// 03.04

Sincronização entre Processos

A sincronização serve para organizar a execução dos processos, evitando conflitos no acesso a recursos compartilhados.

⚠ Deadlock (impasse): Dois ou mais processos ficam esperando um pelo outro indefinidamente — nenhum consegue continuar, causando travamento total do fluxo de execução.

04 — Confiabilidade
// 04.01

Tolerância a Falhas

É a capacidade do sistema continuar funcionando mesmo quando algo dá errado, como a queda de um servidor ou falha de rede.

🔁 Redirecionamento de Tarefas

O sistema redistribui automaticamente as tarefas para outras máquinas disponíveis.

⚡ Operação Parcial

O sistema continua funcionando mesmo com falhas parciais, sem interromper o serviço por completo.

05 — Clusters & Grid
// 05.01

Cluster de Computadores

Um cluster é um conjunto de computadores ligados entre si que trabalham como se fossem um único sistema — de forma transparente para o usuário final.

  • As tarefas são divididas entre várias máquinas de forma coordenada
  • O processamento acontece em paralelo, tornando tudo mais rápido
  • Escala horizontalmente conforme a demanda cresce
// 05.02

Cluster Beowulf

O Cluster Beowulf é um tipo de cluster feito com computadores comuns (como PCs), conectados em rede para trabalhar juntos — sem necessidade de hardware especializado ou proprietário.

💰 Baixo Custo

Usa hardware simples e acessível, tornando a alta performance computacional viável para mais organizações.

⚡ Alto Desempenho

Foca em entregar grande capacidade de processamento com recursos comuns, unindo o melhor dos dois mundos.

// 05.03

Cluster Computing vs Grid Computing

Dois paradigmas de computação distribuída com abordagens e escalas distintas:

🏢 Cluster Computing

Computadores próximos, geralmente no mesmo local, trabalhando juntos de forma organizada.

✔Mais centralizado, com controle único
✔Gerenciado por uma única entidade
✔Baixa latência entre os nós

🌐 Grid Computing

Computadores espalhados em diferentes locais, conectados pela internet.

✔Altamente distribuído geograficamente
✔Envolve múltiplas organizações participantes
✘Maior heterogeneidade e complexidade de gestão

Diferença principal: O cluster é centralizado e controlado por uma única entidade; o Grid é mais distribuído e envolve vários participantes de diferentes organizações conectados pela internet.
